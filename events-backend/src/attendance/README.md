# Event Attendance System

This module provides a comprehensive attendance tracking system for events using QR codes and manual check-in methods. It supports both admin-managed check-ins and user self-service check-ins.

## Features

### 1. QR Code Generation
- Each user profile has a unique QR code
- QR codes can be generated for attendance purposes
- QR codes contain user information and can be scanned by external devices

### 2. Attendance Check-in Methods

#### A. QR Code Scanning (Recommended)
- **Endpoint**: `POST /api/attendance/check-in-qr-code`
- **Method**: Scan user's QR code to automatically check them in
- **Benefits**: Fast, accurate, reduces human error
- **Requirements**: User must be registered for the event

#### B. Manual Check-in (Backup)
- **Endpoint**: `POST /api/attendance/manual-check-in`
- **Method**: Admin manually enters user ID to check them in
- **Use Case**: When QR codes fail or for users without QR codes

### 3. External QR Code Scanner Integration

For external QR code scanning devices (like handheld scanners connected to laptops):

1. **Scan QR Code**: Use external scanner to read user's QR code
2. **Send to Backend**: Send the scanned data to `POST /api/attendance/scan-qr-code`
3. **Check-in**: Use the response to determine if user can be checked in
4. **Complete Check-in**: Use `POST /api/attendance/check-in-qr-code` to finalize

### 4. User Self-Service Check-in (NEW)

Attendees can check themselves in by scanning event QR codes generated by organizers:

- **Self Check-in**: Users scan event QR codes to check themselves in
- **Auto-registration**: For free events, unregistered users can be automatically registered
- **Payment Redirect**: For paid events, unregistered users are redirected to payment
- **Contact Exchange**: Users can exchange contact information by scanning each other's QR codes
- **Exhibitor Stamps**: Attendees collect stamps by visiting exhibition booths

### 5. Event QR Code Management (NEW)

Organizers can create and manage different types of QR codes:

- **Check-in QR Codes**: For attendance tracking
- **Contact Exchange QR Codes**: For networking
- **Exhibitor Stamp QR Codes**: For booth visits
- **Configurable Settings**: Validity periods, scan limits, auto-registration options

## API Endpoints

### Public Endpoints (No Authentication Required)

#### Scan QR Code for Attendance
```
POST /api/attendance/scan-qr-code
Body: { "qrCodeId": "string", "eventId": "string" }
```
- Returns user information and registration status
- Can be used by external scanners
- No authentication required

### User Self-Service Endpoints (Require User Authentication)

#### Self Check-in via Event QR Code
```
POST /api/attendance/self-check-in
Body: {
  "eventQRCodeId": "string",
  "eventId": "string",
  "location": "string (optional)",
  "notes": "string (optional)"
}
```

#### Get User Exhibitor Stamps

## Workflow for External Scanners

1. **Setup**: Connect external QR code scanner to laptop/computer
2. **Scan**: Use scanner to read user's QR code
3. **Process**: Send scan data to backend via `POST /api/attendance/scan-qr-code`
4. **Verify**: Check response to see if user is registered and can check in
5. **Check-in**: If eligible, use `POST /api/attendance/check-in-qr-code` to complete check-in

## Workflow for User Self-Service Check-in

1. **Organizer Setup**: Admin creates event QR codes with appropriate settings
2. **QR Code Placement**: Place QR codes at event locations (entrance, booths, etc.)
3. **User Scan**: Attendees scan QR codes using their event app
4. **Auto-processing**: System automatically handles registration and check-in
5. **Feedback**: User receives confirmation or payment redirect as needed

## Workflow for Contact Exchange

1. **User A**: Scans User B's personal QR code
2. **Contact Exchange**: System creates contact exchange record
3. **Networking**: Both users now have each other's contact information
4. **History**: Users can view all their contact exchanges

## Workflow for Exhibitor Stamp Collection

1. **Booth Setup**: Organizer creates exhibitor stamp QR codes for each booth
2. **User Visit**: Attendee visits booth and scans QR code
3. **Stamp Collection**: System records the visit and adds to user's stamp collection
4. **Progress Tracking**: User can view total stamps collected and booth visits

## Error Handling

The system provides clear error messages for common scenarios:

- **User not registered**: "User is not registered for this event. Cannot check in."
- **Already checked in**: "User is already checked in for this event."
- **Invalid QR code**: "Invalid QR code. Please check your code and try again."
- **Event not found**: "Event not found"
- **User not found**: "User not found"

## Data Model

### EventAttendance Entity
- Links users to events with attendance status
- Tracks check-in/check-out times
- Records check-in method (QR code, manual, admin)
- Stores location and notes
- Links to registration records
- Links to event QR codes (for self check-ins)

### EventQRCode Entity (NEW)
- Organizer-generated QR codes for events
- Configurable types: check-in, contact exchange, exhibitor stamp
- Validity periods and scan limits
- Auto-registration and payment redirect settings
- Location and description information

### ContactExchange Entity (NEW)
- Records contact exchanges between users
- Links scanner and scanned user
- Event context and location tracking
- Prevents duplicate exchanges

### ExhibitorStamp Entity (NEW)
- Records booth visits by attendees
- Links to event QR codes and booths
- Tracks stamp collection progress
- Prevents duplicate stamps from same booth

### Attendance Statuses
- `checked_in`: User is currently at the event
- `checked_out`: User has left the event
- `no_show`: User was registered but didn't attend

### Check-in Methods
- `qr_code`: Scanned via QR code
- `manual`: Manual entry by admin
- `admin`: Admin-initiated check-in

### Event QR Code Types
- `check_in`: For attendance tracking
- `contact_exchange`: For networking
- `exhibitor_stamp`: For booth visits

### Event QR Code Statuses
- `active`: QR code is available for use
- `inactive`: QR code is temporarily disabled
- `expired`: QR code has passed its validity period
